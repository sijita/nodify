# Adaptador — OpenCode

> Agente de terminal de IA por SST. Web: [opencode.ai](https://opencode.ai) · Repo: [github.com/sst/opencode](https://github.com/sst/opencode) · Schema: [opencode.ai/config.json](https://opencode.ai/config.json)
> Fecha de verificación: 2026-07-02 (schema y archivos reales de la máquina cruzados).

## Ubicaciones de config (scope global/usuario)

OpenCode carga y **fusiona** (no reemplaza) varias fuentes:

| Scope | Ruta | Notas |
|-------|------|-------|
| **Global / usuario (macOS/Linux)** | `~/.config/opencode/opencode.json(c)` | XDG (`$XDG_CONFIG_HOME/opencode/`). **Scope principal de Nodify.** |
| **Global / usuario (Windows)** | `%APPDATA%\opencode\opencode.json` | ⚠️ sin confirmar; `%ProgramData%\opencode` para config administrada. |
| **Proyecto** | `opencode.json(c)` en la raíz | Sube por el árbol hasta la raíz del worktree git. |
| **Custom** | env `OPENCODE_CONFIG` | |
| **Inline** | env `OPENCODE_CONFIG_CONTENT` | |

**Precedencia** (posterior sobrescribe claves en conflicto; las demás se conservan): remote (org) → global (usuario) → `OPENCODE_CONFIG` → proyecto → `OPENCODE_CONFIG_CONTENT` → gestionada → managed prefs de macOS.

**Credenciales / API keys** NO viven en `opencode.json`: se guardan en `~/.local/share/opencode/auth.json` (gestionado por `opencode auth login` / `/connect`). **Nodify no debe tocar ese archivo.**

Fuente: [opencode.ai/docs/config](https://opencode.ai/docs/config).

## Formato de archivo

- **JSON y JSONC.** Se aceptan `opencode.json` y `opencode.jsonc`.
- **Comentarios soportados: SÍ.** El schema raíz declara `"allowComments": true` y `"allowTrailingCommas": true` (verificado en [opencode.ai/config.json](https://opencode.ai/config.json)) — comentarios `//`, `/* */` y comas colgantes válidos incluso en `.json`.
- Se recomienda `"$schema": "https://opencode.ai/config.json"` como primera clave.
- **Implicación:** editar con parser que **preserve comentarios y formato** (`jsonc-parser` con ediciones quirúrgicas), nunca `JSON.parse`+`JSON.stringify`.

## MCPs

`mcp` es un objeto donde **cada clave es el nombre del servidor**. Dos transportes discriminados por `type` (schema `$defs.McpLocalConfig` / `McpRemoteConfig`, `additionalProperties: false`).

### Local (`type: "local"`)
| Campo | Tipo | Req. | Notas |
|-------|------|------|-------|
| `type` | `"local"` | ✅ | |
| `command` | **`string[]`** | ✅ | **Array**: comando + args juntos. NO string. |
| `cwd` | `string` | | Directorio de trabajo. |
| `environment` | `{ [k]: string }` | | Env vars. **Se llama `environment`, no `env`.** |
| `enabled` | `boolean` | | Habilita/deshabilita al arrancar. |
| `timeout` | `integer` (ms) | | Default 5000. |

```json
{
  "mcp": {
    "my-local-mcp": {
      "type": "local",
      "command": ["npx", "-y", "@modelcontextprotocol/server-everything"],
      "environment": { "MY_ENV_VAR": "value" },
      "enabled": true,
      "timeout": 5000
    }
  }
}
```

### Remote (`type: "remote"`)
| Campo | Tipo | Req. | Notas |
|-------|------|------|-------|
| `type` | `"remote"` | ✅ | |
| `url` | `string` | ✅ | Endpoint. |
| `headers` | `{ [k]: string }` | | Cabeceras (p.ej. `Authorization`). |
| `oauth` | `McpOAuthConfig \| false` | | `false` desactiva autodetección OAuth. |
| `enabled` | `boolean` | | |
| `timeout` | `integer` (ms) | | Default 5000. |

```json
{
  "mcp": {
    "context7": {
      "type": "remote",
      "url": "https://mcp.context7.com/mcp",
      "enabled": true,
      "headers": { "CONTEXT7_API_KEY": "••••" }
    }
  }
}
```

**Flag `enabled`:** booleano opcional por servidor → ideal para toggles en UI sin borrar la entrada.

### ⚠️ Hallazgo crítico: los archivos reales de esta máquina NO cumplen el schema

Usan forma estilo **Claude Code / Cursor**, no la de OpenCode:

```jsonc
"firecrawl-mcp": {
  "type": "local",
  "command": "npx",                 // ⚠️ string; el schema exige string[]
  "args": ["-y", "firecrawl-mcp"],  // ⚠️ 'args' NO existe en el schema OpenCode
  "env": { "FIRECRAWL_API_KEY": "••••" },  // ⚠️ debería ser 'environment'
  "enabled": true
}
```

Forma canónica OpenCode correcta:
```json
"firecrawl-mcp": {
  "type": "local",
  "command": ["npx", "-y", "firecrawl-mcp"],
  "environment": { "FIRECRAWL_API_KEY": "••••" },
  "enabled": true
}
```

**Nodify debe normalizar `command`(string)+`args` → `command`(array)** y **`env` → `environment`** al escribir OpenCode válido. Estos archivos los genera el plugin `opencode-synced` (comentario `// Generated by opencode-synced`); no asumir que un `opencode.json` existente ya es válido.

Fuente: [opencode.ai/docs/mcp-servers](https://opencode.ai/docs/mcp-servers).

## Skills

Dos mecanismos conviven:

### 1. Skills nativas (recomendado)
Soporte de primera clase con herramienta `skill` integrada. Cada skill = directorio con `SKILL.md` + frontmatter YAML.

Frontmatter: `name` (req., regex `^[a-z0-9]+(-[a-z0-9]+)*$`, = nombre de dir), `description` (req., 1–1024). Opcionales: `license`, `compatibility`, `metadata`.

**Descubrimiento** (sube desde cwd; la primera con un nombre gana → proyecto sobrescribe usuario):
- Proyecto: `.opencode/skills/<name>/SKILL.md`, `.claude/skills/<name>/SKILL.md`, `.agents/skills/<name>/SKILL.md`
- Global/usuario: `~/.config/opencode/skills/*/SKILL.md`

> Nota: OpenCode **lee también `.claude/skills/`** → los skills de Claude son en buena medida portables a OpenCode.

**Habilitar/deshabilitar:**
- Permisos con wildcard: `{ "permission": { "skill": { "*": "allow", "internal-*": "deny" } } }`
- Desactivar la herramienta: `{ "agent": { "plan": { "tools": { "skill": false } } } }`
- Campo raíz `skills` (rutas/URLs a carpetas). ⚠️ semántica exacta sin confirmar.

### 2. Plugin `opencode-agent-skills` (modo mantenimiento)
Se activa en el array `plugin`: `{ "plugin": ["opencode-synced", "opencode-agent-skills"] }`. Elementos = nombres de paquete npm (opcional versión pineada). Lee de `.opencode/skills/`, `.claude/skills/`, `~/.config/opencode/skills/`, `~/.claude/skills/`. Enable/disable = añadir/quitar del array (no hay flag por plugin).

Fuentes: [opencode.ai/docs/skills](https://opencode.ai/docs/skills), [opencode-agent-skills](https://github.com/joshuadavidthomas/opencode-agent-skills).

## Config general

### Proveedores y modelos (`provider`)
Objeto por ID de proveedor. 75+ proveedores vía **AI SDK** + catálogo **Models.dev**. Ejemplo (verificado contra la máquina):
```json
{
  "provider": {
    "featherless": {
      "name": "Featherless",
      "npm": "@ai-sdk/openai-compatible",
      "options": { "baseURL": "https://api.featherless.ai/v1" },
      "models": { "moonshotai/Kimi-K2.7-Code": { "name": "Kimi K2.7" } }
    }
  }
}
```
Campos: `name`, `npm`, `api`, `id`, `env` (`string[]`), `whitelist`/`blacklist`, `options` (`apiKey`, `baseURL`, `timeout`, ...), `models`.
**API keys:** preferente vía `opencode auth login` → `auth.json`, o `options.apiKey` con sintaxis env `"{env:MY_KEY}"`. Evitar keys en claro en `opencode.json`.

### Modelo por defecto
- `model` raíz, formato `"provider/model-id"` (p.ej. `"anthropic/claude-sonnet-4"`).
- `small_model`: tareas ligeras (títulos, resúmenes).

### Filtrado de proveedores
- `disabled_providers` / `enabled_providers` (`string[]`) raíz. En esta máquina: `"disabled_providers": ["featherless"]`.

### Reglas / system prompt (`AGENTS.md` + `instructions`)
- **`AGENTS.md`**: instrucciones inyectadas. Proyecto: raíz. Global: `~/.config/opencode/AGENTS.md`. Búsqueda: locales (subiendo) → global → fallback `~/.claude/CLAUDE.md`.
- **`instructions`** (raíz, `string[]`): rutas/globs a archivos extra; soporta URLs remotas. Se **combinan** con los `AGENTS.md`.

### Otros campos raíz (awareness)
`shell`, `logLevel`, `server`, `command`, `skills`, `watcher`, `snapshot`, `plugin`, `share`, `autoshare`, `autoupdate`, `default_agent`, `username`, `mode`, `agent`, `formatter`, `lsp`, `layout`, `permission`, `tools`, `attachment`, `enterprise`, `tool_output`, `compaction`, `experimental`.

Fuentes: [providers](https://opencode.ai/docs/providers), [rules](https://opencode.ai/docs/rules).

## Campos a preservar (unknown-field preservation)

El schema raíz es `additionalProperties: false` (no admite claves desconocidas a nivel raíz), pero Nodify **debe preservar todo lo que no gestione** porque otras herramientas escriben el archivo (`opencode-synced` lo genera) y hay ~35 campos raíz.

**Preservar íntegramente:**
- `$schema`.
- **Comentarios y comas colgantes** (`.jsonc` y `.json`) → parser que preserve trivia, nunca `JSON.parse`+`stringify`.
- Array `plugin` completo (`opencode-synced`, `opencode-agent-skills`).
- Bloque `provider` completo (custom, `npm`, `options.baseURL`, `models`).
- `disabled_providers`/`enabled_providers`, `model`, `small_model`, `default_agent`.
- `agent`, `permission`, `tools`, `instructions`, `skills`, `mode`, `formatter`, `lsp`, `mcp` (entradas no editadas).
- Sub-campos de cada MCP no gestionados (`cwd`, `timeout`, `oauth`, `headers`).

**Estrategia:** ediciones quirúrgicas por clave (surgical merge) sobre el AST, tocando solo rutas gestionadas; round-trip idempotente. **Nunca** reescribir el archivo entero desde un modelo parcial.
